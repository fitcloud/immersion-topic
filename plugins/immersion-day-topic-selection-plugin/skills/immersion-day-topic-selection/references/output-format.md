# 출력 형식

최종 답변 또는 HTML 콘텐츠 계획을 만들 때 이 형식을 사용한다.

## 최종 답변 필수 섹션

```text
# Immersion Day 주제 선정 결과

## 1. 선정 목적
- 주요 목적:
- 교육 대상:
- 교육 시간:
- 고려한 제약사항:
- 회사 맥락 반영 수준:

## 2. 회사 맥락
| 항목 | 확인한 내용 | 주제 선정에 미친 영향 |

## 3. 기존 FitCloud 교육 확인
| 기존 교육 | URL | 유사성 판단 | 반영 방식 |
기본 확인 범위는 올해 진행한 FitCloud Immersion Day이다. 작년 이전 주제는 사용자가 명시적으로 제외하지 않으면 자동 제외하지 않는다.
확인 범위와 상세 페이지 확인 개수를 함께 적는다.

## 4. 조사 기준 및 참고 소스
| 구분 | 확인 목적 | 실제 사용 근거 |
AWS Knowledge Base MCP 사용 가능 여부를 명시한다. 사용할 수 없었다면 AWS 공식 웹 fallback이라고 표시한다.

## 5. 1차 후보 5-7개
| 번호 | 후보 주제 | 핵심 아이디어 | 관련 AWS 서비스 | 회사 맥락 연결 | 기존 교육 중복 위험 |

## 6. 평가 기준
`evaluation-rubric.md`의 1/3/5 점수 기준을 요약한다.

## 7. 평가표
| 후보 | 최신성 | 고객 문제 가설 신뢰도 | 실습 | AWS | 차별성 | 발표 | 오퍼링 | 재사용 | 회사 적합성 | 총점 |

평가표 아래에는 후보별 기준 단위 설명표를 포함한다.

| 후보 | 기준 | 점수 | 왜 이 점수인가 | 왜 더 높은 점수가 아닌가 |

규칙:
- 모든 기준 점수에는 이유가 있어야 한다.
- 1-4점은 더 높은 점수가 아닌 이유를 설명한다.
- 5점은 "5점 기준을 충족함"처럼 명확히 쓴다.
- 근거, 제약, 부족한 asset과 연결되지 않은 모호한 표현은 피한다.

## 8. 산정 이유 요약
| 후보 | 높은 점수 이유 | 감점/리스크 이유 | 5점이 아닌 핵심 이유 | 회사 적합성 | 핵심 근거 |

## 9. 최종 추천 3개
각 후보에 다음을 포함한다.
- 한 줄 설명
- 권장 대상과 시간
- 선정 이유
- 점수 산정 이유
- 기준별 점수 이유와 감점 이유
- 관련 AWS 서비스
- AWS Knowledge Base MCP 근거 또는 미사용 사유
- AWS 공식 근거
- YouTube / 외부 AI Engineering 근거
- FitCloud 기존 교육과의 차이
- 회사 맥락 근거
- 실습 시나리오
- 서비스 오퍼링 가능성
- 리스크

## 10. 최종 비교 요약
| 구분 | 후보 1 | 후보 2 | 후보 3 |

## 11. 투표용 요약
1. 주제명:
   - 핵심:
   - 장점:
   - 우려:
   - 산정 이유:
   - 회사 적합성:
   - 추천 이유:
```

## HTML 요구사항

HTML이 요청되면:

- `assets/voting-template.html`을 기본 템플릿으로 사용한다.
- 최종 3개 카드 포함
- 점수표 포함
- 점수 산정 이유 요약 포함
- 회사 적합성 근거 포함
- 근거 자료 요약 포함
- service offering badge 포함
- radio 기반 투표 선택 포함
- localStorage save/restore 포함
- 투표 결과 복사와 초기화 action 포함
- static HTML은 단일 브라우저 저장만 제공한다는 안내 포함
